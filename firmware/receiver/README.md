# 수신기 펌웨어

보드와 출력 회로가 확정되기 전까지 실제 핀과 라이브러리를 고정하지 않습니다.

## 책임

- 무선 링크 초기화와 재연결
- 패킷 검증
- 손실, 중복과 순서 역전 통계
- 지터 버퍼
- 손실 프레임 처리
- 디코딩 또는 PCM 출력
- 안전한 음소거와 볼륨
- 배터리와 상태 표시

## 예상 구조

- app: 전체 상태와 실행 흐름
- transport: 무선 수신
- protocol: 패킷 해석과 검증
- jitter_buffer: 재생 순서와 대기 시간 관리
- decoder: 코덱 또는 PCM 처리
- audio_output: I2S와 출력 장치
- diagnostics: 로그와 통계
- platform: 보드 종속 코드

## 구현 전 필요한 정보

- 보드와 무선 방식
- DAC, 앰프 또는 코덱 모델
- 출력 샘플 형식
- 지터 버퍼 초기 깊이
- 언더런과 손실 처리 정책
- 안전한 최대 음량과 mute 제어

## 첫 구현 순서

1. M0 패킷 수신과 sequence 통계
2. 주기적인 상태 로그
3. 링크 차단과 재연결 시험
4. M1 시험 payload 검증
5. 시험 톤 출력
6. 실제 오디오 출력
