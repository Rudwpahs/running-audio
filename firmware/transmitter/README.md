# 송신기 펌웨어

보드와 통신 방식이 확정되기 전까지 실제 보드 종속 코드는 작성하지 않습니다.

## 책임

- 입력 장치 초기화
- 일정한 샘플 수집
- 버퍼와 오버런 관리
- 프로토콜 헤더 생성
- 무선 송신
- 상태와 성능 통계
- 안전한 시작과 종료

## 예상 구조

- app: 전체 상태와 실행 흐름
- audio_input: 오디오 입력
- buffering: 샘플 큐와 오버런 처리
- protocol: 헤더와 payload 생성
- transport: 무선 전송
- diagnostics: 로그와 통계
- platform: 보드 종속 코드

## 빌드 시스템 후보

- PlatformIO
- ESP-IDF
- Arduino core 기반 PlatformIO

최종 선택은 보드, 오디오 드라이버와 자동 시험 가능성을 비교해 의사결정 기록에 남깁니다.

## 구현 전 필요한 정보

- 보드 정확한 모델과 메모리
- 입력 방식
- 무선 방식과 최대 payload
- 샘플레이트, 비트 깊이와 채널 수
- 패킷 주기와 버퍼 크기
- 핀 배치와 전원 조건

## 첫 구현 순서

1. M0 증가 카운터 패킷 송신
2. 송신 성공, 실패와 큐 깊이 통계
3. M1 시험 데이터 또는 PCM payload
4. 실제 스마트폰 입력 연결
