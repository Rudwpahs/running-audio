# 요구사항

Running Audio MVP가 무엇을 해야 하고 어떤 기준으로 성공을 판단할지 정의합니다. 수치는 최종 제품 사양이 아니라 초기 검증 목표입니다.

## 기능 요구사항

| ID | 요구사항 | 초기 수용 기준 |
|---|---|---|
| FR-01 | 스마트폰 오디오를 송신기가 입력받아야 한다. | 테스트 톤과 음악 신호 수집 성공 |
| FR-02 | 송신기는 입력을 패킷으로 만들어야 한다. | sequence number와 timestamp 포함 |
| FR-03 | 수신기는 손실·중복·순서 역전을 판별해야 한다. | 관련 카운터 제공 |
| FR-04 | 수신기는 오디오 출력으로 전달해야 한다. | 테스트 톤 또는 음성 재생 |
| FR-05 | 연결·오류 상태를 확인할 수 있어야 한다. | LED 또는 직렬 로그 제공 |
| FR-06 | 링크 단절 후 재연결해야 한다. | 송신기·수신기 재부팅 시험 |
| FR-07 | 부팅·오류 시 갑작스러운 최대 음량이 없어야 한다. | 무음·뮤트 처리 |
| FR-08 | 향후 다중 수신기 확장을 막지 않아야 한다. | stream ID와 broadcast 개념 유지 |
| FR-09 | 실험 결과를 재현할 수 있어야 한다. | 보드·커밋·설정·거리·전원 기록 |

## 비기능 요구사항

| ID | 항목 | 초기 목표 | 측정 방법 |
|---|---|---|---|
| NFR-01 | 종단 간 지연 | 150ms 이하를 1차 목표로 측정 | 원 신호와 출력 파형 비교 |
| NFR-02 | 실내 거리 | 장애물 없는 10m | 1m, 5m, 10m 단계 시험 |
| NFR-03 | 벽 통과 | 일반 실내 벽 1개 조건 확인 | 동일 설정 반복 시험 |
| NFR-04 | 패킷 손실 | 10분 손실률 기록 | sequence 차이 집계 |
| NFR-05 | 연속 작동 | 최소 30분 | 정전압 또는 완충 배터리 시험 |
| NFR-06 | 부품비 | 송신기+수신기 실제 구매가 산출 | 배송비 포함·제외 구분 |
| NFR-07 | 수신기 무게 | 보드 선정 후 상한 설정 | 배터리·케이스 포함 실측 |
| NFR-08 | 발열 | 피부 접촉 예상 부위 온도 기록 | 10분 간격 측정 |
| NFR-09 | 안정성 | 잘못된 패킷·버퍼 고갈로 재부팅되지 않음 | 오류 주입과 장시간 시험 |
| NFR-10 | 개인정보 | 음원과 장치 비밀정보를 저장하지 않음 | 로그·파일 검토 |

## 제약조건

- 초기 개발은 학생 예산과 가정 내 시험 장비를 전제로 합니다.
- 운동장 Wi-Fi, 이동통신, 클라우드를 필수 조건으로 삼지 않습니다.
- 보드와 통신 방식이 확정되기 전에는 핀과 라이브러리를 고정하지 않습니다.
- 배터리 착용 시험은 보호회로와 발열 검증 이후에 실시합니다.

## 초기 범위 밖

- 상용 오디오 제품 수준의 음질
- 방수·방진·낙하 인증
- 의료 또는 안전 장비 용도
- 대규모 다중 사용자 운영
- 음원 배포·저장 기능
