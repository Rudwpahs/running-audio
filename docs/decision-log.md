# 기술 의사결정 기록

결정되지 않은 내용을 코드가 먼저 고정하지 않도록 관리합니다.

상태는 검토 중, 제안, 확정, 대체됨으로 구분합니다.

## ADR-001 스마트폰 오디오 입력 방식

- 날짜: 2026-07-07
- 상태: 검토 중
- 문제: 스마트폰 오디오를 송신기에 안정적으로 입력하는 방법이 필요함
- 후보: 3.5mm 아날로그, USB-C 디지털, Bluetooth 수신, 스마트폰 앱과 UDP
- 결정: 미정
- 평가 기준: 지연, Android와 iOS 호환성, 부품비, 개발 난이도, 충전 포트 점유, 음질
- 재검토 조건: 보드 후보와 1차 구매 예산 확정 후

## ADR-002 무선 전송 방식

- 날짜: 2026-07-07
- 상태: 검토 중
- 문제: 외부 인터넷 없이 데이터를 전송해야 함
- 후보: ESP-NOW, Wi-Fi SoftAP와 UDP, Bluetooth Classic, BLE Audio, 전용 2.4GHz 오디오 모듈
- 결정: 미정
- 평가 기준: 지연, 손실, 다중 수신, 구현 난이도, 전력, 가격
- 재검토 조건: M0 패킷 시험 결과 확보 후

## ADR-003 보드 선정

- 날짜: 2026-07-07
- 상태: 검토 중
- 문제: I2S, 무선, 메모리와 전력 요구를 만족하는 저가 보드 필요
- 후보: ESP32-S3, 일반 ESP32, 오디오 개발보드, 전용 송수신 모듈
- 결정: 미정
- 평가 기준: 실제 구매가, 정확한 모델, PSRAM, I2S, USB 기능, 소비전력, 문서와 예제
- 재검토 조건: 입력과 무선 방식 후보가 축소된 후

## ADR-004 오디오 형식과 코덱

- 날짜: 2026-07-07
- 상태: 검토 중
- 문제: 대역폭, 지연과 CPU 사용량의 균형 필요
- 후보: PCM 16-bit mono, ADPCM, Opus 계열, 전용 모듈 내장 코덱
- 결정: 미정
- 평가 기준: 프레임 길이, 알고리즘 지연, 구현 가능성, 손실 내성, 라이선스
- 재검토 조건: M0 대역폭과 손실 측정 후

## ADR-005 전원 방식

- 날짜: 2026-07-07
- 상태: 검토 중
- 후보: USB 전원, 보호회로 포함 Li-Po, 18650 모듈, 보조배터리

## ADR-006 수신기 출력 방식

- 날짜: 2026-07-07
- 상태: 검토 중
- 후보: 3.5mm 이어폰, I2S DAC와 앰프, 골전도 트랜스듀서, 기성 Bluetooth 이어폰 연계
- 결정: 미정
- 평가 기준: 지연, 음량 안전, 무게, 효율, 회로 난이도, 주변음 인지
