# T3-S3 MVSR H594-01 연결 기준

## 1. 두 보드 공통

1. 보드에 맞는 2.4 GHz 안테나를 먼저 연결합니다.
2. 안테나 없이 무선 펌웨어를 실행하지 않습니다.
3. 첫 시험은 배터리 없이 USB 전원만 사용합니다.
4. 한 대에는 TX 펌웨어, 다른 한 대에는 RX 펌웨어를 올립니다.

## 2. 송신기

외부 오디오 ADC나 마이크 배선은 없습니다.

```text
Windows PC -> USB 데이터 케이블 -> T3-S3 MVSR TX
```

PC 재생음은 Windows loopback으로 캡처하거나, 먼저 440 Hz 테스트 톤을
생성해 보냅니다.

## 3. 수신기

MVSR 보드에는 MAX98357A가 이미 실장되어 있으므로 첫 온보드 스피커 시험에
점퍼선이나 외장 앰프가 필요하지 않습니다.

| 내부 신호 | ESP32-S3 GPIO |
|---|---:|
| MAX98357A BCLK | 40 |
| MAX98357A LRCLK / WS | 41 |
| MAX98357A DATA | 39 |
| MAX98357A SD_MODE | 38 |

펌웨어는 GPIO38을 HIGH로 설정하고 I2S 포트 1로 16 kHz stereo 복제 데이터를
출력합니다.

## 4. SX1280 RF 스위치

H594-01의 실제 RF 경로에는 다음 제어 핀이 필요합니다.

| 기능 | ESP32-S3 GPIO |
|---|---:|
| RF TX enable | 10 |
| RF RX enable | 21 |

펌웨어는 RadioLib의 RF switch 제어에 `RX=21, TX=10`을 등록합니다.

## 5. 골전도 트랜스듀서 #1674

첫 시험은 반드시 MVSR 온보드 스피커로 끝낸 뒤 진행합니다.

- 보드 실물에서 스피커 출력 커넥터 규격과 극성을 확인합니다.
- 확인 전에는 2핀 커넥터를 추가 구매하거나 임의 납땜하지 않습니다.
- 앰프의 두 스피커 출력 중 어느 쪽도 GND나 GPIO에 연결하지 않습니다.
- 첫 전기 시험에서는 트랜스듀서를 머리에서 떼어 놓습니다.
- PC volume 0.10과 펌웨어 limiter 4096 상태에서 시작합니다.
- 비정상 발열·냄새·왜곡이 있으면 즉시 USB를 분리합니다.
