# 전송 프로토콜 초안

특정 무선 기술을 확정하지 않고 공통 패킷 요구만 정의합니다.

## 헤더 후보

| 필드 | 목적 |
|---|---|
| magic | 다른 데이터와 구분 |
| protocol_version | 버전 관리 |
| header_length | 확장 헤더 지원 |
| stream_id | 스트림 구분 |
| receiver_id 또는 broadcast | 1대1 또는 1대다 표시 |
| sequence_number | 손실, 중복, 순서 역전 탐지 |
| timestamp | 재생 시점과 지연 분석 |
| codec | PCM, ADPCM 등 형식 표시 |
| sample_rate | 샘플레이트 표시 |
| channels | mono 또는 stereo |
| flags | 시작, 종료, 무음, 제어 표시 |
| payload_length | 길이 검증 |
| payload | 오디오 또는 시험 데이터 |
| checksum | 선택적 손상 검증 |

실제 바이트 크기와 순서는 선택한 무선 기술의 최대 payload를 확인한 뒤 결정합니다.

## 기본 규칙

- sequence number는 패킷마다 증가합니다.
- 값이 최대치에서 0으로 돌아가는 경우를 고려합니다.
- timestamp 단위는 샘플 카운터 또는 마이크로초 중 하나로 통일합니다.
- 송수신 시계가 동기화되지 않으면 timestamp만으로 절대 지연을 판단하지 않습니다.

## 패킷 크기

작은 프레임은 지연과 단일 손실 영향을 줄이지만 헤더 비율과 송신 횟수가 증가합니다. 큰 프레임은 효율적이지만 지연과 손실 영향이 커집니다. M0에서 안정적인 크기와 전송률을 측정합니다.

## 지터 버퍼

- 초기에는 단순한 고정 깊이 버퍼로 시작합니다.
- buffer depth, 언더런과 대기시간을 기록합니다.
- 측정 결과를 바탕으로 적응형 버퍼를 검토합니다.

## 손실 처리 후보

- 무음 삽입
- 직전 프레임 반복
- 단순 보간
- 코덱의 손실 은닉 기능

초기 구현은 단순하고 갑작스러운 큰 소리를 만들지 않는 방식을 선택합니다.

## 재전송 후보

- 재전송 없음
- 제한적인 누락 요청
- 무선 링크가 제공하는 자동 재전송

최종 방식은 평균 지연뿐 아니라 최대 지연과 손실률을 함께 측정해 결정합니다.
