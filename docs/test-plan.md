# 시험 계획

모든 결과는 tests/test-log-template.md를 복사해 기록합니다.

## 공통 측정

- 송신 패킷과 수신 패킷 수
- 손실, 중복, 순서 역전
- 평균 지연, 최대 지연, 지터
- 버퍼 언더런과 오버런
- 재연결 시간
- 전압, 전류, 온도
- 주관적 끊김과 음질

## 시험 항목

### 책상 1m
- 목적: 기본 송수신 확인
- 조건: 가시거리와 USB 전원
- 절차: 10분 연속 전송
- 결과: 리셋, 손실, 지연과 끊김 기록

### 방 내부
- 목적: 일반 실내 환경 확인
- 조건: 3m에서 5m, 가구 존재
- 절차: 위치 3곳에서 각 10분
- 결과: 조건별 차이 기록

### 벽 1개
- 목적: 장애물 영향 확인
- 조건: 문 열림과 닫힘을 구분
- 절차: 동일 설정으로 반복
- 결과: 손실과 지연 차이 기록

### 복도 거리
- 목적: 실내 거리 한계 탐색
- 절차: 5m 간격으로 거리를 늘림
- 결과: 품질 저하 시작점과 연결 한계 기록

### 지연
- 목적: 종단 간 지연 측정
- 절차: 클릭 신호의 원본과 수신 출력을 함께 녹음
- 결과: 30회 이상 측정해 평균과 최대값 산출

### 장시간과 재연결
- 30분 연속 재생
- 송신기 재부팅 후 복구 시간
- 수신기 재부팅 후 복구 시간
- 일시적 링크 차단 후 복구 여부

### 전원과 온도
- 시작과 종료 전압
- 평균과 최대 전류
- 10분 간격 표면 온도
- 케이스 유무에 따른 차이

## M0 패킷 시험

오디오 입력 전 고정 크기 데이터로 링크만 검증합니다.

- payload: 증가 카운터와 고정 패턴
- 헤더: magic, version, sequence, timestamp, payload length
- 전송률: 낮은 주기부터 단계적으로 증가
- 출력: 송신 수, 수신 수, 손실, 중복, 순서 역전, 지연
