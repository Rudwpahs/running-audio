# PR1 실제 프로토타입 v3

## 1. 이번 개발 관문의 목적

이 단계는 학습용 시뮬레이션이 아니라, 두 대의 실제 T3-S3 MVSR 보드로 다음
종단간 경로가 성립하는지 검증하는 단계입니다.

```text
PC PCM -> USB 프레임 -> TX -> FLRC 3조각 -> RX 재조립
-> 40 ms 지터 시작 버퍼 -> 온보드 MAX98357A -> 스피커
```

시장성, 다중 수신기, 배터리, 케이스, 야외 장거리 시험은 이 관문을 통과한
후에 다룹니다.

## 2. 고정 하드웨어

- LILYGO T3-S3 MVSR H594-01, SX1280 2.4 GHz without PA: 2대
- 각 보드용 2.4 GHz 안테나: 2개
- USB-A to USB-C 데이터 케이블: 2개
- MVSR 온보드 MAX98357A와 온보드 스피커
- Adafruit 골전도 트랜스듀서 #1674: 온보드 스피커 성공 후 출력 교체 시험
- microSD 32 GB: 보유하되 이번 PC-USB 첫 관문에서는 사용하지 않음

외장 MAX98357A, 별도 ESP32, 점퍼선, 브레드보드, 배터리, 라우터는 첫 관문에
사용하지 않습니다.

## 3. 고정 오디오·무선 규격

- PCM: signed 16-bit little-endian, mono
- 샘플레이트: 16 kHz
- 논리 프레임: 160 samples / 320 bytes / 10 ms
- USB 프레임: 20바이트 헤더 + 320바이트 PCM
- RF: SX1280 FLRC, 2410.5 MHz
- FLRC bit rate: 650 kbps
- FLRC coding rate: 3/4
- 출력 설정: 3 dBm
- RF sync word: `50 52 50 31` (`PRP1`)
- RF 하드웨어 CRC: 2바이트
- 애플리케이션 CRC: 논리 PCM 프레임 전체 CRC32
- 재전송: 첫 실시간 오디오 관문에서는 사용하지 않음

## 4. 수신 동작

1. 동일 프레임의 세 조각을 순서와 무관하게 수집합니다.
2. 중복 조각은 한 번만 반영합니다.
3. 더 새로운 프레임이 도착할 때 이전 프레임이 불완전하면 폐기합니다.
4. 세 조각이 모이면 전체 PCM CRC32를 다시 계산합니다.
5. 정상 프레임만 오디오 큐로 전달합니다.
6. 출력 시점에 프레임이 없으면 10 ms 무음을 넣어 실시간 시계를 유지합니다.
7. 수신기에서 샘플 절댓값을 4096으로 제한해 첫 출력 레벨을 낮춥니다.
8. PC sender를 재실행하면 새 32비트 stream ID로 지터 큐와 sequence 기준을
   다시 맞춥니다.

## 5. 합격 기준

다음 조건을 모두 만족해야 실제 PoC PASS입니다.

1. TX와 RX가 각각 빌드·플래시됩니다.
2. 두 보드가 모두 `READY` 로그를 냅니다.
3. 440 Hz 저음량 테스트 톤이 온보드 스피커에서 식별됩니다.
4. Windows 재생음이 알아들을 수 있는 수준으로 연속 출력됩니다.
5. 1 m 2분 시험에서 보드 리셋과 지속적 큐 오버플로가 없습니다.
6. 실내 5 m 10분 시험에서 TX/RX 카운터와 RSSI를 기록합니다.
7. 10분 동안 예상치 못한 보드 리셋이 0회입니다.
8. 온보드 스피커 관문 뒤, 올바른 출력 커넥터를 확인한 상태에서만
   골전도 트랜스듀서를 시험합니다.

## 6. 아직 주장할 수 없는 것

- 실제 RF 도달거리
- 야외·공원 범위
- 다중 수신기 수용량
- 배터리 사용시간
- 골전도 착용 음질·누음·열 안전성
- KC 적합성 또는 판매 가능 상태

이 항목들은 실측 전까지 가설로만 유지합니다.
