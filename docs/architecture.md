# 시스템 구조

## 전체 흐름

스마트폰 오디오 → 입력 인터페이스 → 송신기 → 근거리 무선 링크 → 수신기 → DAC 또는 앰프 → 이어폰·골전도 출력

입력 인터페이스, 무선 방식, 보드와 출력 회로는 아직 확정하지 않습니다.

## 데이터 흐름

1. 스마트폰이 음악 또는 테스트 톤을 재생합니다.
2. 입력부가 신호를 디지털 샘플로 제공합니다.
3. 송신기가 일정 시간 단위로 버퍼링합니다.
4. 선택한 형식으로 payload를 만듭니다.
5. sequence number와 timestamp를 포함한 패킷을 전송합니다.
6. 수신기는 형식, 길이, 순서와 손실을 검증합니다.
7. 지터 버퍼가 도착 시간 변동을 흡수합니다.
8. 복원한 오디오를 출력 장치로 전달합니다.

## 블록별 책임

### 입력부
- 스마트폰 호환성 관리
- 아날로그 레벨 또는 디지털 형식 변환
- 앱 방식일 경우 권한과 백그라운드 동작 관리

### 송신기
- 샘플 수집과 시간 기준 유지
- 입력 버퍼 오버런 감지
- 패킷 생성과 송신
- 연결, 오류, 전송 통계

### 무선 링크
- 연결 또는 피어 등록
- payload 크기와 전송률 제약 확인
- 채널 혼잡, 거리와 전력 영향 측정

### 수신기
- 패킷 검증
- 손실, 중복, 순서 역전 감지
- 지터 버퍼와 언더런 처리
- 디코딩과 안전한 오디오 출력

### 전원부
- 보호, 충전, 과전류, 단락과 발열 관리
- 실제 소비전류 측정
- 저전압 시 반복 재부팅과 큰 소리 방지

## 주요 오류 지점

| 구간 | 오류 | 관측 | 초기 대응 |
|---|---|---|---|
| 입력 | 샘플 누락, 형식 불일치 | 입력 카운터 | 무음과 오류 로그 |
| 송신 버퍼 | 오버런 | 최대 버퍼 깊이 | 폐기 정책 비교 |
| 무선 | 손실, 중복, 순서 역전 | sequence 통계 | 손실 처리 방식 비교 |
| 수신 버퍼 | 언더런, 지연 증가 | buffer depth | 무음, 반복, 보간 비교 |
| 출력 | 클릭음, 과대 음량 | 청감과 파형 | 페이드인, 리미터, 뮤트 |
| 전원 | 전압 강하, 발열 | 전압, 전류, 온도 | 보호회로와 저전압 종료 |

## 확장 방향

- 하나의 송신기에서 여러 수신기로 전송
- 수신기별 상태와 볼륨
- 복수 스트림 또는 채널
- PC 기반 송수신 시뮬레이터
- 측정과 설정 도구

확장 기능은 최초 1대1 링크와 측정 체계가 안정된 뒤 진행합니다.
