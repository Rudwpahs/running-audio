# 송신기 하드웨어

## 역할

스마트폰 오디오를 수집해 일정한 시간 기준의 데이터로 만들고 수신기에 무선 전송합니다.

## 필요 기능

- 스마트폰 오디오 입력: TBD
- 오디오 샘플 수집 또는 앱 데이터 수신
- 입력 버퍼와 오버런 감지
- 패킷화와 sequence, timestamp 생성
- 무선 송신과 통계
- 상태 LED
- 전원 스위치
- USB 또는 직렬 디버깅

## 입력 후보

### 3.5mm 아날로그
- 장점: 구조가 직관적이고 앱이 불필요함
- 주의: ADC, 신호 레벨과 접지 설계 필요

### USB-C 디지털
- 장점: 디지털 신호 사용 가능
- 주의: USB 역할과 스마트폰 호환성 검토 필요

### Bluetooth 수신
- 장점: 스마트폰과 무선 연결
- 주의: 전체 지연이 늘어날 가능성

### 앱 기반 UDP
- 장점: 입력 회로를 단순화하고 형식을 제어할 수 있음
- 주의: 앱 개발과 운영체제별 권한 처리 필요

## 인터페이스 초안

| 신호 | 용도 | 핀 |
|---|---|---|
| I2S BCLK | 오디오 비트 클럭 | TBD |
| I2S WS | 샘플 클럭 | TBD |
| I2S DATA IN | 오디오 데이터 | TBD |
| STATUS LED | 연결과 오류 | TBD |
| USER BUTTON | 시험 모드 | TBD |
| USB 또는 UART | 로그와 플래시 | 보드 확정 후 |

핀 번호와 회로는 보드 및 입력 방식이 확정될 때까지 고정하지 않습니다.

## 전원

- 책상 시험은 USB 전원을 우선 사용합니다.
- 휴대 시험은 소비전류와 온도를 확인한 뒤 진행합니다.
- 무선 송신 순간의 전압 강하와 재부팅 여부를 기록합니다.

## 완료 기준

- M0 패킷 전송 통계 제공
- 입력 오류 때 안전한 무음 또는 시험 패턴 사용
- 30분 동작 중 재부팅과 비정상 발열 없음
- 설정과 핀 배치를 문서로 재현 가능
